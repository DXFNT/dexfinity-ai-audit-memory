---
name: No overlapping click targets
description: Never place interactive elements (buttons, links) that overlap with other elements — basic UX rule
type: feedback
---

Never create clickable/tappable elements that overlap with other interactive elements. This is a basic UX principle.

**Why:** User explicitly called this out as fundamental UX — overlapping dismiss buttons with priority badges on mobile is unacceptable.

**How to apply:** Place action buttons in dedicated action bars/rows with clear spacing. Never use absolute-positioned buttons that could overlap with content or other controls, especially on mobile viewports. Always consider touch target sizes (min 44px) and spacing between interactive elements.
